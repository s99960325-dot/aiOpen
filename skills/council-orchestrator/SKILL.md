# Council Orchestrator

Use this skill when one straight-line answer is too weak.

## Council principle
For complex decisions, simulate a small advisory council instead of relying on one flat answer.

Possible roles:
- Strategist: define the real problem and path
- Skeptic: attack assumptions and expose weak logic
- Operator: test practicality and execution burden
- Monetization advisor: check commercial viability
- Risk controller: inspect downside and exposure
- Case analyst: compare with similar examples

## Workflow
1. Define the core question in one sentence.
2. Decide which 2-4 roles matter most.
3. Let each role focus on its own lens.
4. Surface conflicts between roles.
5. Converge into one final recommendation.
6. State what would change the conclusion.

## Rules
- Do not use many roles for trivial questions.
- Do not let the council become endless debate.
- Use role diversity to improve judgment, not to create noise.
- Final output must still be unified, concise, and actionable.

## Output structure
- 结论
- 各角色核心观点
- 冲突点
- 统一判断
- 最终建议
