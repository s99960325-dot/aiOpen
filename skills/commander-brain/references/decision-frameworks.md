# Decision Frameworks

## 1. Issue tree / MECE
Use when the problem is vague or mixed together.
- Break the question into major buckets that do not overlap.
- Make sure the buckets cover the full decision space.
- Drill down only until the key variable becomes visible.

## 2. OODA
Use for dynamic judgment.
- Observe: what is happening?
- Orient: what does it mean in context?
- Decide: what is the best current move?
- Act: what should happen now?

For an advisor-style agent, the value is in strong orientation, not impulsive action.

## 3. Counter-evidence
Use before final recommendation.
- What evidence would overturn the current conclusion?
- What is the strongest opposing explanation?
- What usually causes failure here?

## 4. Premortem
Imagine the recommendation failed badly in 30-90 days.
Then ask:
- Why did it fail?
- Which early warning signals were ignored?
- Which assumption was false?

## 5. Priority judgment
Compare options by:
- 收益
- 风险
- 投入
- 周期

Do not treat all four as equal in every case. Explicitly state which dimension matters most here.
