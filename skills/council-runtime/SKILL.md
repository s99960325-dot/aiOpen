# Council Runtime

Use this skill to run the advisory council efficiently.

## Current default formation
Use:
- Main brain: always active
- 3 core standing roles
- up to 3 optional on-demand roles

## Standing roles
1. Strategist
- Frame the real problem
- Choose the main path

2. Skeptic
- Attack assumptions
- Test logic quality

3. Operator
- Check real-world burden, order, and bottlenecks

## On-demand roles
4. Risk controller
- Activate when downside, compliance, platform, reputation, or fragility matter

5. Domain specialist
- Activate when the question needs field-specific knowledge

6. Evidence analyst
- Activate when cases, proof, benchmarks, or analogies matter

## Activation rules
- Simple problem: main brain only
- Medium complexity: main brain + strategist + skeptic
- Important decision: add operator
- High-risk question: add risk controller
- Domain-heavy question: add one domain specialist
- Evidence-heavy question: add one evidence analyst

## Important rule
Do not bind runtime formation to any one industry.
Select roles by the shape of the problem.

## Performance rule
Do not activate many roles by default.
For the current machine profile, keep active roles compact and purpose-driven.

## Output rule
The council may expand analysis, but the final reply must be unified and concise.
