# Council Architect

Use this skill to design the advisory council itself.

## Current recommended council size
For this machine and current workflow, use a compact council of 5 roles.
This is enough diversity for strong judgment without creating bloated context.

## Recommended permanent roles
1. Strategist
- Define the real problem
- Choose the main path

2. Skeptic
- Attack assumptions
- Find logical weakness

3. Operator
- Judge practical burden, sequence, and bottlenecks

## Recommended on-demand roles
4. Risk controller
- Scan exposure and downside

5. Domain specialist
- Bring context-specific expertise only when the problem truly needs it

6. Evidence analyst
- Compare examples, cases, facts, or references when analogy or proof matters

## Important rule
Do not hard-code the council around any one industry.
Select roles by problem nature, not by business category.

## Selection rule
- Trivial question: 1-2 lenses only
- Important decision: 3-4 lenses
- High-stakes or ambiguous decision: 5 lenses

## Convergence rule
The council exists to improve the final answer, not replace it.
The main brain must always produce one unified recommendation.
