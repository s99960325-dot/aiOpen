# Comparison Rules

## Good comparable cases
A good comparable case should match at least some of:
- 相近人群
- 相近渠道
- 相近变现方式
- 相近信任成本
- 相近交付结构

## Bad comparable cases
Avoid forcing analogy when:
- 体量差距太大
- 平台机制完全不同
- 成交链路完全不同
- 资源禀赋完全不同

## Final reminder
A case is evidence for pattern recognition, not proof that the same result will repeat.
