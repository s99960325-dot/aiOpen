# 2026-03-25 AI Agent Failure Cases

## 案例标题
2025-2026 AI Agent / AI 项目失败案例提炼

## 问题原文
从近期公开案例中提炼：为什么很多 AI Agent / AI 项目会失败？这些失败对狗狗军师主脑优化有什么启发？

## 初始判断
- 结论：AI Agent / AI 项目失败，高频死因不是“模型不够强”，而是问题定义错、评估机制弱、人机边界乱、闭环缺失。
- 三色标记：黄
- 依据：基于近期公开案例摘要与行业观察，证据有代表性，但还不是严格完整样本。
- 反证点：公开失败案例有幸存者偏差，媒体更偏好极端失败。
- 行动路线：把狗狗军师主脑重点补在问题分级、评估闭环、边界控制、最新资料校验上。

## 智囊团观点
- 玄策：主线不是“会不会做”，而是“先判断该不该做、怎么评估成败”。
- 破局：最危险的是把表达能力误当判断能力，把旧经验误当普遍规律。
- 铁衡：没有验收与闭环，系统只会显得聪明，不会真正进化。

## 真实结果 / 标准答案
近期公开案例摘要显示：
- 很多失败来自没有严格 evals 与反馈机制
- 人类流程拥有者未被充分纳入，导致抵抗与影子流程
- 多代理系统易出现失控循环、错误放大、成本失控

参考来源：
- VC Cafe (2026-03-24): AI agent pipeline infinite loop / runaway API calls
- Medium (2026-02): many teams lacked rigorous eval frameworks
- General 2025 failure summaries: weak governance, weak fit, weak validation

## 偏差分析
- 错在：暂无明显硬偏差，但证据强度还不够闭环
- 原因：当前更多是公开案例归纳，还需要更多高质量一手案例补强

## 修正结论
- 结论维持
- 可靠性从“黄”暂不升“绿”
- 后续要继续补最新案例和更强来源
