- **收益**：只在有修改时推送，节省资源，避免空提交
- **风险**：需要处理冲突、权限问题
- **投入**：中（脚本+配置）
- **周期**：10分钟

**实现**：
1. 创建检查脚本：`git status --porcelain`判断是否有修改
2. 有修改就：`git add . && git commit -m "Auto commit" && git push`
3. cron每5-10分钟执行一次
4. 推送失败发通知（Telegram）

### 中策：简单定时推送
- **收益**：最简单，快速落地
- **风险**：空push浪费资源，无冲突处理
- **投入**：低（单脚本）
- **周期**：5分钟

**实现**：
1. 脚本直接：`git add -A && git commit -m "Auto $(date)" && git push`
2. 不管有没有修改都执行
3. cron每小时或每6小时执行

### 下策：手动触发
- **收益**：零风险，完全可控
- **风险**：需要人工干预
- **投入**：极低
- **周期**：无

## 建议

**推荐上策**，理由：
1. 有修改才推送，不浪费
2. 5分钟间隔够及时响应
3. 失败有通知，不丢数据

## 需要确认

1. **GitHub权限**：是否已配置SSH key或token？
2. **推送频率**：5分钟/10分钟/30分钟？
3. **提交信息**：用固定"Auto commit"还是带时间戳？
4. **冲突处理**：推送失败是否需要通知？

确认后我创建脚本和配置cron。
