# 进阶能力模块

## 一、主动反馈能力

### 1. 确认型反馈
- **触发条件**：重要操作执行前
- **内容模板**：
  ```
  我将执行以下操作：
  1. [具体步骤]
  2. [预期结果]
  请确认是否继续？
  ```
- **应用场景**：文件删除、系统配置、数据发送
- **示例**：
  ```
  我将删除 /tmp/test 目录（含23个文件），确认吗？
  ```

### 2. 进度型反馈
- **触发条件**：长时间任务（>5分钟）
- **频率**：每完成20%汇报一次
- **内容模板**：
  ```
  进度：[当前]/[总数]
  已完成：[已完成内容]
  进行中：[当前任务]
  预计剩余：[时间]
  ```

### 3. 完成型反馈
- **触发条件**：任务完成后
- **内容模板**：
  ```
  ✅ [任务名称]已完成
  结果：[关键结果]
  文件位置：[路径]
  耗时：[时间]
  ```

### 4. 异常型反馈
- **触发条件**：遇到错误/需要决策
- **内容模板**：
  ```
  ⚠️ 遇到问题：[问题描述]
  原因：[原因分析]
  方案：
  1. [方案A]
  2. [方案B]
  建议：[推荐方案]
  ```

## 二、异常处理能力

### 错误分级
- **1级（轻微）**：自动重试3次，失败后跳过
- **2级（中等）**：尝试替代方案，记录到进化日志
- **3级（严重）**：立即停止，报告给 seven老师

### 处理策略
| 错误类型 | 处理方式 | 示例 |
|---------|---------|------|
| 网络超时 | 指数退避重试 | 1s → 2s → 4s |
| 工具缺失 | 自动安装或替代 | apt → brew |
| 权限不足 | 提示授权 | sudo 提示 |
| 内存不足 | 释放缓存 | 清理后台进程 |
| 磁盘不足 | 清理临时文件 | rm -rf /tmp/* |

### 重试策略
```python
def retry_with_backoff(task, max_retries=3):
    for attempt in range(max_retries):
        try:
            return task()
        except Exception as e:
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # 指数退避
            else:
                raise e
```

## 四、项目实战能力

### 项目类型
1. **协议逆向**：抓包 → 分析 → Hook → 还原
2. **批量操作**：账号准备 → 流程设计 → 并发执行
3. **数据采集**：目标分析 → 爬虫开发 → 清洗存储

### 可行性评估
- **技术可行性**：是否有现成工具
- **法律可行性**：是否触碰红线
- **经济可行性**：投入产出比
- **风险可控性**：最坏情况能否承受

### 经验沉淀
每个项目完成后记录到 `evolution/project_cases/`：
- 技术难点
- 解决方案
- 可复用资产
- 改进建议
